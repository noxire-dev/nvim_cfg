# nvim-min Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the existing Neovim config at `~/AppData/Local/nvim` with a minimal, hand-rolled, ThePrimeagen-style config (~12 plugin spec files, ~22 packages), while safely preserving the old config on the GitHub remote and as a local `nvim.bak/` directory.

**Architecture:** lazy.nvim plugin manager with one spec file per plugin under `lua/mvim/lazy/`. Single `init.lua` chains three requires (`set`, `remap`, `lazy_init`). LSP via mason + lspconfig + nvim-cmp. No UI sugar (no dashboard, no notifier, no statusline customization, no mini suite, no snacks).

**Tech Stack:** Neovim 0.10+, lazy.nvim, telescope.nvim, nvim-treesitter, nvim-lspconfig, mason.nvim, nvim-cmp, conform.nvim, gitsigns.nvim, vim-fugitive, harpoon (v2), undotree, zen-mode, which-key, copilot.lua, rose-pine + tokyonight + kanagawa colorschemes.

**Verification approach:** This is a personal Neovim config, not application code — there is no test runner. "Tests" are manual verification by launching `nvim` and confirming each feature behaves as described. Each task includes the exact commands and expected output. Frequent commits give bisect surface if something later breaks.

**Pre-flight prerequisites** — verify before starting Task 1:

```

where rg          # ripgrep, for telescope live_grep
```

If any are missing, stop and install them first. Do not skip the C compiler check — TreeSitter parser builds fail silently on Windows without one.

Mason + mason-lspconfig + nvim-lspconfig + nvim-cmp + LuaSnip. Single file for the whole LSP/completion stack.

**Files:**

- [ ] **Step 1: Write `lua/mvim/lazy/lsp.lua`**

Exact contents:

```lua
return {
  {
    "neovim/nvim-lspconfig",
    dependencies = {
      "mason-org/mason.nvim",
      "mason-org/mason-lspconfig.nvim",
      "hrsh7th/cmp-nvim-lsp",
    },
    config = function()
      require("mason").setup()
      require("mason-lspconfig").setup({
        ensure_installed = { "lua_ls", "gopls", "pyright", "ruff", "ts_ls" },
      })

      local capabilities = vim.tbl_deep_extend(
        "force",
        vim.lsp.protocol.make_client_capabilities(),
        require("cmp_nvim_lsp").default_capabilities()
      )

      local lspconfig = require("lspconfig")
      for _, server in ipairs({ "lua_ls", "gopls", "pyright", "ruff", "ts_ls" }) do
        lspconfig[server].setup({ capabilities = capabilities })
      end

      vim.api.nvim_create_autocmd("LspAttach", {
        callback = function(ev)
          local opts = { buffer = ev.buf, silent = true }
          local function map(mode, lhs, rhs, desc)
            vim.keymap.set(mode, lhs, rhs, vim.tbl_extend("force", opts, { desc = desc }))
          end

          map("n", "gd", vim.lsp.buf.definition, "Go to definition")
          map("n", "gD", vim.lsp.buf.declaration, "Go to declaration")
          map("n", "gr", vim.lsp.buf.references, "References")
          map("n", "gi", vim.lsp.buf.implementation, "Implementation")
          map("n", "K", vim.lsp.buf.hover, "Hover docs")
          map("n", "<leader>vrn", vim.lsp.buf.rename, "Rename symbol")
          map({ "n", "v" }, "<leader>vca", vim.lsp.buf.code_action, "Code action")
          map("n", "<leader>vd", vim.diagnostic.open_float, "Diagnostic float")
          map("n", "[d", function() vim.diagnostic.jump({ count = -1 }) end, "Prev diagnostic")
          map("n", "]d", function() vim.diagnostic.jump({ count = 1 }) end, "Next diagnostic")
        end,
      })
    end,
  },
  {
    "hrsh7th/nvim-cmp",
    event = "InsertEnter",
    dependencies = {
      "hrsh7th/cmp-nvim-lsp",
      "L3MON4D3/LuaSnip",
      "saadparwaiz1/cmp_luasnip",
    },
    config = function()
      local cmp = require("cmp")
      cmp.setup({
        snippet = {
          expand = function(args)
            require("luasnip").lsp_expand(args.body)
          end,
        },
        mapping = cmp.mapping.preset.insert({
          ["<C-Space>"] = cmp.mapping.complete(),
          ["<C-n>"] = cmp.mapping.select_next_item(),
          ["<C-p>"] = cmp.mapping.select_prev_item(),
          ["<CR>"] = cmp.mapping.confirm({ select = true }),
          ["<C-e>"] = cmp.mapping.abort(),
        }),
        sources = cmp.config.sources({
          { name = "nvim_lsp" },
          { name = "luasnip" },
        }),
      })
    end,
  },
}
```

## Notes for the implementing engineer

- **Frequent commits are non-negotiable.** Every task commits at its end. If a step in the middle of a task feels self-contained, an extra commit is fine.
- **If lazy.nvim cache gets weird**, `:Lazy clean` then `:Lazy sync` usually fixes it. If a plugin won't load, `:Lazy reload <name>` is the lighter hammer.
- **Mason on Windows can be slow on cold start.** Don't bail out at 30 seconds — give it 5 minutes for the first run of Task 7.
- **Treesitter parser failures = missing C compiler.** `:checkhealth nvim-treesitter` will tell you. Do not silently skip — syntax highlighting depends on this.
- **Don't add features the plan doesn't specify.** No "while I'm here" plugins (no autopairs, no indent guides, no statusline). The user explicitly opted for minimal — respect that.
- **Don't migrate old keymaps from `nvim.bak/`** unless this plan lists them. The old `<leader>sf` etc. are intentionally gone; their replacements are `<leader>pf` etc.
